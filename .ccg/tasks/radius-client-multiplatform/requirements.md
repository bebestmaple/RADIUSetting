# 需求文档 - RADIUS 认证客户端

## 项目目标

开发跨平台 RADIUS/802.1X 认证客户端，支持大规模部署，简化用户操作流程。

## 目标平台

### Windows（GUI 必须）
- Windows XP（可接受额外依赖，如 .NET Framework 3.5）
- Windows 7
- Windows 10
- Windows 11

### macOS（GUI）
- macOS 10.12+ （支持主流版本）

### Linux（GUI + CLI 脚本）
- Ubuntu Desktop/Server
- Debian
- CentOS 7

## 核心功能

### 1. 认证管理

**认证协议**：EAP-PEAP（Protected EAP）
- 外层：TLS 加密隧道
- 内层：MS-CHAPv2（用户名/密码）

**网络配置**：
- **不需要用户配置** RADIUS 服务器地址/端口（由交换机/AP 侧配置，802.1X 协商自动完成）
- 软件仅配置客户端侧的 802.1X 认证参数

**证书验证**：
- 首次连接：弹窗显示 RADIUS 服务器证书信息，用户确认后导入系统证书存储
- 后续连接：自动使用已信任证书，无需再次确认
- 证书信息展示：颁发者、有效期、指纹

**凭据管理**：
- 用户每次启动时**手动输入**用户名/密码
- **不保存密码**（安全考虑）

**一键操作**：
- 启用认证：输入凭据 → 点击"连接" → 自动配置 802.1X → 触发认证
- 禁用认证：点击"断开" → 恢复网络默认配置

### 2. 诊断功能

**目标**：快速排查客户端配置问题，而非网络侧问题。

**Windows 诊断**：
- ✅ 检查必要的系统服务状态：
  - `Wired AutoConfig` (dot3svc) — **有线 802.1X 必需，默认禁用**
  - `WLAN AutoConfig` (WlanSvc) — 无线 802.1X 必需
  - `EapHost` — EAP 认证服务
- ✅ **自动修复功能**：检测到服务未运行时，提供"自动启动"按钮
- ✅ 检查网络适配器 802.1X 配置状态
- ✅ 显示最近认证日志（事件查看器中的 802.1X 相关条目）

**macOS 诊断**：
- ✅ 检查 `eapolclient` 服务状态
- ✅ 检查网络接口 802.1X 配置
- ✅ 显示系统日志中的认证记录

**Linux 诊断**：
- ✅ 检查 `wpa_supplicant` / `NetworkManager` 服务状态
- ✅ 检查必要的包是否安装：
  - Debian/Ubuntu: `wpasupplicant`, `network-manager`
  - CentOS: `wpa_supplicant`, `NetworkManager`
- ✅ 检查网络接口配置

**诊断输出**：
- 状态灯：🟢 正常 / 🟡 警告 / 🔴 错误
- 问题描述 + 修复建议（如"服务未启动，点击自动修复"）

### 3. 状态监控

- 实时显示认证状态：未连接 / 认证中 / 已连接 / 认证失败
- 显示当前网络接口信息（IP、MAC、连接时长）
- 托盘图标 + 右键菜单快捷操作

## 部署方案

### 分发方式
- **Portable 包**（绿色版）：解压即用，无需安装
- 适合大规模部署（通过共享文件夹/U盘/脚本分发）

### 配置管理
- **配置文件**（`config.json` 或 `config.ini`）预设：
  - 认证方式：PEAP
  - 证书验证策略：首次确认
  - 日志级别
  - （可选）组织信息、技术支持联系方式
- 用户**无需修改配置**（除非特殊需求）

### 权限要求
- Windows：需要管理员权限（修改网络配置、导入证书）
- macOS/Linux：需要 root/sudo 权限

## 技术约束

### 跨平台兼容性
- Windows XP/7：可接受额外依赖（.NET Framework、VC++ Runtime）
- 所有平台**必须提供 GUI**（命令行仅作为补充，不可替代 GUI）

### 系统 API 调用
- Windows：使用 `netsh` / Windows Native WiFi API / EAP Host API
- macOS：使用 `networksetup` / `eapolclient` / Security Framework
- Linux：使用 `wpa_supplicant` / `NetworkManager` D-Bus API

### 安全要求
- 密码不保存到磁盘（仅保存在内存，进程退出后清除）
- 证书导入需要用户明确确认
- 配置文件不包含敏感信息

## 验收标准

1. ✅ 在 9 个目标平台上成功启用 PEAP 认证
2. ✅ 用户操作不超过 3 步（运行软件 → 输入凭据 → 点击连接）
3. ✅ 诊断功能能检测出常见配置问题（服务未启动、包缺失）
4. ✅ 证书首次连接时正确弹窗，用户确认后不再提示
5. ✅ Portable 包在目标系统上无需安装即可运行（依赖项可单独安装）

## 非功能需求

- **性能**：认证触发响应时间 < 3 秒
- **可靠性**：异常情况下能回滚网络配置（不留残留配置）
- **可维护性**：代码模块化，平台特定代码隔离
- **文档**：提供用户手册（截图 + 步骤）和管理员部署指南

## 风险与挑战

1. **旧系统 API 兼容性**（WinXP/Win7 API 已废弃）
2. **Linux 发行版差异**（Debian 系 vs RedHat 系网络管理工具不同）
3. **权限提升**（需要优雅处理 UAC/sudo 提示）
4. **多语言 GUI 框架选择**（需权衡包体积、性能、开发效率）
